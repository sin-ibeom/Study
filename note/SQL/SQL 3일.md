1. 집합 연산자 (Set Operators)
   두 개 이상의 SELECT문의 조회 결과를 하나로 합쳐서 출력함.
   * 주의: 합치려는 두 SELECT문의 열 개수와 자료형(Type)이 일치해야 함.

   (1) UNION (합집합 - 중복 제거)
       두 쿼리의 결과 중 중복된 데이터는 한 번만 출력함.
       * 내부적으로 중복을 제거하는 정렬 과정이 있어 UNION ALL보다 속도가 느릴 수 있음.

       <형식>
       SELECT [열1], [열2] FROM [테이블A]
       UNION
       SELECT [열1], [열2] FROM [테이블B];

   (2) UNION ALL (합집합 - 중복 포함)
       두 쿼리의 결과를 중복 여부와 상관없이 모두 출력함.
       * 단순 합치기이므로 속도가 빠름.

       <형식>
       SELECT [열1], [열2] FROM [테이블A]
       UNION ALL
       SELECT [열1], [열2] FROM [테이블B];

2. 오라클 주요 함수 : 대소문자 변환

   (1) UPPER / LOWER 함수
       * UPPER(문자열) : 괄호 안의 문자 데이터를 모두 '대문자'로 변환
       * LOWER(문자열) : 괄호 안의 문자 데이터를 모두 '소문자'로 변환

   (2) 실무 활용 (검색 기능 구현 시 필수)
       데이터베이스에 저장된 값이 대문자인지 소문자인지 모를 때,
       양쪽을 모두 대문자(혹은 소문자)로 통일시켜 비교함.

       <예시: 사원 이름에 'scott'이 포함된 사람 찾기>
       SELECT *
       FROM   EMP
       WHERE  UPPER(ENAME) LIKE UPPER('%scott%');
       
       -> 결과: 'SCOTT', 'Scott', 'scott' 등 대소문자 상관없이 모두 조회됨.

3. 오라클 주요 함수 : 문자열 길이

   (1) LENGTH 함수
       문자열의 '글자 수(길이)'를 반환함. (한글, 영문 상관없이 글자 수)

       <예시: 이름이 5글자 이상인 사원 조회>
       SELECT ENAME, LENGTH(ENAME)
       FROM   EMP
       WHERE  LENGTH(ENAME) >= 5;

   (2) LENGTHB 함수
       문자열이 차지하는 '바이트(Byte) 수'를 반환함.
       
       * 영문/숫자: 1 Byte
       * 한글: 오라클 설정에 따라 2 Byte 또는 3 Byte (UTF-8 등)
       
       <비교>
       LENGTH('한글')  -> 2 (글자 수)
       LENGTHB('한글') -> 6 (UTF-8 기준 바이트 수)


1. SQL 단일행 함수 (문자열 처리)

   (1) 대소문자 변환 함수
       * UPPER(문자열): 모든 문자를 대문자로 변환
       * LOWER(문자열): 모든 문자를 소문자로 변환
       * INITCAP(문자열): 첫 글자만 대문자로 변환 (실무 활용도 낮음)
       
       <실무 활용 팁>
       검색 기능 구현 시, 사용자 입력값과 DB 저장값의 대소문자가 다를 수 있음.
       이때 양쪽 모두 UPPER() 또는 LOWER()를 적용하여 비교하면 대소문자 구분 없이 검색 가능.
       예) WHERE UPPER(ENAME) LIKE UPPER('%adam%')

   (2) 문자열 길이 함수
       * LENGTH(문자열): 문자의 개수(길이)를 반환 (한글, 영문 무관하게 글자 수)
       * LENGTHB(문자열): 문자열의 바이트(Byte) 수를 반환
         - 영문/숫자: 1 Byte
         - 한글: 인코딩 방식에 따라 2~3 Byte (UTF-8 등)

   (3) DUAL 테이블
       * 오라클에서 자체 제공하는 더미(Dummy) 테이블.
       * 특정 테이블의 데이터와 관계없이 계산식이나 함수의 결과를 테스트해볼 때 사용.
       예) SELECT UPPER('abc') FROM DUAL;

2. 컴퓨터 기초 지식 (용량 단위)

   (1) 비트(Bit)와 바이트(Byte)
       * Bit (b): 컴퓨터의 최소 단위 (0 또는 1)
       * Byte (B): 8 Bit = 1 Byte (문자 하나를 저장하는 기본 단위)

   (2) 용량 단위 체계 (1024배씩 증가)
       * 1 KB (Kilobyte) = 1024 Byte
       * 1 MB (Megabyte) = 1024 KB
       * 1 GB (Gigabyte) = 1024 MB
       * 1 TB (Terabyte) = 1024 GB

   (3) 통신 속도 단위 (bps)
       * bps (bits per second): 초당 전송 비트 수
       * 우리가 흔히 쓰는 Byte 단위로 환산하려면 '나누기 8'을 해야 함.
       예) 100Mbps 인터넷 속도 -> 실제 다운로드 속도는 약 12.5MB/s

3. Git (버전 관리 시스템) 기초

   (1) Git의 주요 기능
       * 버전 관리: 파일의 변경 이력을 저장하고 이전 시점으로 되돌리기 가능.
       * 협업: 여러 명이 동시에 하나의 프로젝트를 작업할 때 충돌 관리 및 코드 병합.

   (2) Git 작업 흐름 (Workflow)
       1. 작업 파일 수정
       2. 스테이지(Stage)에 올리기: 커밋할 파일 선택
       3. 커밋(Commit): 변경 사항 확정 (로컬 저장소)
       4. 푸시(Push): 원격 저장소(GitHub 등)에 업로드

   (3) 다운로드 및 동기화
       * Clone: 원격 저장소 내용을 통째로 복제 (최초 1회)
       * Fetch: 원격 저장소의 변경 사항 확인 (다운로드 X)
       * Pull: 변경 사항을 다운로드하여 내 코드에 반영 (Fetch + Merge)

   (4) 충돌(Conflict) 해결
       * 같은 파일의 같은 라인을 서로 다르게 수정했을 때 발생.
       * 해결법:
         1. 충돌난 파일 확인 (<<<<, ====, >>>> 표시됨)
         2. 원하는 코드로 수정 (내 코드 or 상대방 코드 or 병합)
         3. 수정 후 다시 커밋 및 푸시
