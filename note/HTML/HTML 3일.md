1. Iframe
========================================
- 개념: 현재 페이지 안에 '창'을 뚫어서 다른 홈페이지를 보여주는 태그
- 특징:
  * 부모(내 페이지)와 자식(불러온 페이지) 관계가 형성됨.
  * 서로 통신(데이터 주고받기)이 가능함.
  * 보안상 취약점이 많아 최근에는 꼭 필요한 경우(지도, 결제창 등) 아니면 지양하는 추세.

1. form
========================================
- 역할: 사용자 입력을 서버로 전송하기 위한 껍데기 태그
- 주의: Form 안에 또 다른 Form을 넣을 수 없음 (중첩 불가).
- 주요 속성:
  (1) action: 데이터를 보낼 목적지(서버 주소). (기본값: 현재 페이지)
  (2) method: 전송 방식 (GET / POST)
      * GET (기본값)
        - 데이터를 주소(URL) 뒤에 붙여서 보냄.
        - 쿼리 스트링(Query String): 주소?name=value&name2=value2 형태
        - 보안에 취약(비번 다 보임), 전송량 제한 있음.
      * POST
        - 데이터를 주소에 안 붙이고, 몸체(Body)에 숨겨서 보냄
        - 대용량 데이터 전송 가능, 보안 유리

1. Input 태그
========================================
(1) 텍스트 입력 계열
  - type="text" (기본값): 일반 글씨 입력
    * value: 초기 입력값
    * placeholder: 입력 전 보여줄 힌트 문구
    * readonly: 읽기 전용 (전송 가능)
    * disabled: 사용 불가 (전송 불가능 ★중요)
  - type="password": 입력 내용이 마스킹(●●●) 처리됨

(2) 버튼 계열 (<input> 태그 사용 시)
  - type="button": 그냥 버튼 (자바스크립트용). value에 버튼명 적음
  - type="reset": Form 내의 모든 내용을 초기값으로 리셋
  - type="submit": Form 내용을 action 주소로 전송

(3) 선택 계열
  - type="checkbox": 다중 선택 가능 (0개 ~ 여러 개)
    * checked: 미리 체크된 상태 설정
  - type="radio": 단일 선택 (여러 개 중 딱 1개만)
    * ★중요: 그룹으로 묶으려면 name 속성값이 같아야 함.
    * checked: 미리 체크된 상태 설정

4. 그 외 form 요소들
========================================
(1) <button> 태그
  - <input type="button">과 비슷하지만 닫는 태그가 있음.
  - <button>버튼명</button>
  - ★주의: type을 안 적으면 기본값이 "submit"임 (엔터 치면 전송됨).

(2) <textarea>
  - 여러 줄의 긴 글을 입력할 때 사용 (메모, 게시글 등).
  - <textarea>초기값은 여기 사이에 적음</textarea>
  - 속성: rows(줄 수), cols(칸 수)로 크기 조절.

(3) <select> 와 <option>
  - 드롭다운 목록(콤보 박스) 만들기.
  - value 속성의 값이 실제 서버로 전송됨.
  - 속성:
    * size: 한 번에 보여줄 목록 개수
    * multiple: 다중 선택 허용 (Ctrl 키 누르고 선택)

5. Label (라벨) - 웹 접근성 필수!
========================================
- 역할: 폼 요소의 이름표. 클릭 영역을 확장해줌 (글씨 눌러도 체크됨).
- 사용법 1 (감싸기 - 추천):
  <label>
      <input type="checkbox"> 동의합니다
  </label>

- 사용법 2 (for + id 연결):
  <label for="user_check">동의합니다</label>
  <input type="checkbox" id="user_check">
